## OOP (Object Orientend Programming)

Great question, Md. Mojnu Miah bhai! тЭдя╕П
Yes тАФ `new Audio()` is actually an example of **Object-Oriented Programming (OOP)** in JavaScript.

Let me explain in simple Bangladeshi-style language. ЁЯСЗ

---

### тЬЕ `new Audio()` is OOP тАФ Why?

- In JavaScript, `Audio` is a **built-in class (constructor function)**.

- When you write:

  ```js
  const audio = new Audio("/sounds/right.mp3");
  ```

  You are creating a **new object (instance)** of the `Audio` class.

- This object comes with **methods and properties**, like:

  - `audio.play()`
  - `audio.pause()`
  - `audio.volume`
  - `audio.loop` etc.

---

### ЁЯУж So it follows OOP concepts:

| Concept            | Explanation                                                     |
| ------------------ | --------------------------------------------------------------- |
| **Class**          | `Audio` is the class (internally defined by browser).           |
| **Object**         | `audio` is the object created from that class.                  |
| **Encapsulation**  | The object has its own internal audio data, methods.            |
| **Method Calling** | We call `audio.play()` or `audio.pause()` like a method in OOP. |

---

### ЁЯза Bonus Example:

You can create **multiple sounds** using multiple `Audio` instances:

```js
const correctSound = new Audio("/sounds/right.mp3");
const wrongSound = new Audio("/sounds/wrong.mp3");

correctSound.play();
```

ThatтАЩs real-world OOP тАФ **you make objects** and **use their methods**.
