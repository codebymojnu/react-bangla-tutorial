If you want to **hardcode data directly into `localStorage`** ‚Äî like in your HTML or JS file ‚Äî you can do it like this:

### ‚úÖ Example: Hardcode object into localStorage

```html
<script>
  // Hardcoded object
  const person = {
    name: "mojnu",
    age: 27,
    gender: "male",
  };

  // Save to localStorage
  localStorage.setItem("person", JSON.stringify(person));
</script>
```

This will store the object as a JSON string in `localStorage` under the key `"person"`.

---

### ‚úÖ If you want to hardcode it **without using a variable**:

```html

    "person",
    JSON.stringify({ name: "mojnu", age: 27, gender: "male" })
  );
</script>
```

To **get the value** you saved in `localStorage`, you can use `localStorage.getItem()` and then parse it using `JSON.parse()`.

### ‚úÖ Example: Get from localStorage

```js
const personString = localStorage.getItem("person");
const person = JSON.parse(personString);

console.log(person);
// Output:
// { name: "mojnu", age: 27, gender: "male" }
```

---

### ‚ö†Ô∏è Why we use `JSON.parse()`?

Because when you save an object to `localStorage`, it‚Äôs stored as a **string**. `JSON.parse()` converts it back to a real object so you can use its properties like:

```js

console.log(person.age); // 27
```
