## তোমার React শেখার প্রথম ধাপ: চলো, শুরু করি\!

React শিখতে এসেছো? দারুণ একটা সিদ্ধান্ত নিয়েছো\! প্রথমে একটু বলে নিই, React কিন্তু দেখতে যতটা কঠিন মনে হয়, ততটা কঠিন একদমই না। সত্যি বলতে, এটা বেশ মজার একটা জিনিস। একবার যদি তুমি এর মূল বিষয়টা ধরে ফেলতে পারো, দেখবে কত সহজে দারুণ সব ওয়েব অ্যাপ্লিকেশন তুমি নিজেই বানিয়ে ফেলতে পারছো।

এই যে আমরা কথা বলছি, এর মধ্যেই তুমি React-এর এমন সব জরুরি জিনিস শিখে ফেলবে, যা তোমার প্রতিদিনের কাজে লাগবে। বিশ্বাস করো, এই একটা লেখা মন দিয়ে পড়লে, _<span style={{color: "blue"}}>React-এর প্রায় ৮০% দরকারি কনসেপ্ট তোমার হাতের মুঠোয় চলে আসবে\!</span>_

<br />

## তো, কী কী শিখতে চলেছি আমরা এই যাত্রায়?

- কম্পোনেন্ট কী, আর কীভাবে একটা কম্পোনেন্টের ভেতরে আরেকটা বসাতে পারি(মানে, নেস্ট করতে হয়), সেটা তুমি নিজে হাতে বানাতে শিখবে।
- তোমার অ্যাপকে সুন্দর করে সাজানোর জন্য মার্কআপ (HTML-এর মতো দেখতে কোড) আর স্টাইল (CSS) কীভাবে যোগ করতে হয়, সেটাও দেখবো।
- অ্যাপের মধ্যে বিভিন্ন তথ্য (ডেটা) কীভাবে দেখাতে হয়, সেটা জানবে।
- কখনো এটা দেখাবো, কখনো ওটা – এই যে কন্ডিশনাল লজিক দিয়ে কিছু দেখানো, কিংবা একটা তালিকা ধরে অনেকগুলো জিনিস কিভাবে দেখানো সম্ভব, এগুলো কীভাবে করতে হয় শিখবে।
- ব্যবহারকারী যখন কোনো বাটনে ক্লিক করবে বা কিছু করবে (মানে, ইভেন্ট), তখন তুমি কীভাবে সাড়া দেবে আর স্ক্রিনে নতুন কিছু দেখাবে (স্ক্রিন আপডেট), সেটাও শিখে ফেলবে।
- সবচেয়ে মজার ব্যাপার, একটা কম্পোনেন্টের তথ্য আরেকটা কম্পোনেন্টের সাথে কীভাবে ডেটা চালাচালি (শেয়ার) করতে হয়, সেটাও জেনে যাবে।

### কম্পোনেন্ট বানাও আর একটার ভেতরে আরেকটা রাখা _Creating and Nesting Components_

আচ্ছা, প্রথমেই আসি কম্পোনেন্ট প্রসঙ্গে। React অ্যাপগুলো তৈরি হয় অনেকগুলো ছোট ছোট _কম্পোনেন্ট_ দিয়ে। এটা একটি দালান কোঠার ইট ভাবতে পারো। প্রত্যেকটা কম্পোনেন্ট হলো তোমার ভিজুয়াল ইউজার ইন্টারফেসের (মানে, ইউজার যে স্ক্রিনে দেখে) একটা অংশ। এই প্রতিটা কম্পোনেন্টে থাকে লজিক লেখার ক্ষমতা। একটা কম্পোনেন্ট যেমন একটা ছোট্ট বাটন হতে পারে, আবার পুরো একটা পেজও একটা কম্পোনেন্ট হতে পারে\!

> React কম্পোনেন্ট বানানো কিন্তু খুব সহজ। এটা আসলে একটা জাভাস্ক্রিপ্ট ফাংশন, যেটা কিছু মার্কআপ (স্ক্রিনে দেখানোর জন্য কোড) তোমাকে রিটার্ন দেবে। দেখো, কী রকম:

```javascript
function MyButton() {
  // এই ফাংশনটা একটা বাটন দেখাচ্ছে
  return <button>আমি একটা বাটন</button>;
}
```

দেখলে? `MyButton` নামে আমরা একটা কম্পোনেন্ট বানিয়ে ফেললাম। এখন তুমি চাইলে এই `MyButton` কম্পোনেন্টটাকে অন্য একটা কম্পোনেন্টের ভেতরে বসিয়ে দিতে পারো। ঠিক যেমন একটা বড় বাক্সের ভেতর আরেকটা ছোট বাক্স রাখা যায়:

```javascript {5}
// ধরা যাক, এটা তোমার পুরো অ্যাপটা
export default function MyApp() {
  return (
    <div>
      <h1>আমার অ্যাপে তোমাকে স্বাগতম!</h1>
      {/* এই দেখো, এখানে MyButton কম্পোনেন্টটা ব্যবহার করেছি */}
      <MyButton />
    </div>
  );
}
```

একটা জিনিস কি খেয়াল করেছো? `<MyButton />` নামটা কিন্তু <span style={{borderBottom: "1px dashed black"}}>বড় হাতের 'M' দিয়ে শুরু হয়েছে।</span> এটাই হলো React কম্পোনেন্ট চেনার একটা সহজ উপায়। **React কম্পোনেন্টের নাম সবসময় বড় হাতের অক্ষর দিয়ে শুরু করতে হবে।** আর সাধারণ HTML ট্যাগ (যেমন `<div>`, `<button>`) কিন্তু ছোট হাতের অক্ষরে লিখতে হয়। এটা একটা ছোট্ট নিয়ম, কিন্তু খুব দরকারি\!

ফলাফলটা কেমন হবে, চলো দেখে নিই:

```javascript
// MyButton.js
function MyButton() {
  return <button>আমি একটা বাটন</button>;
}

// MyApp.js
export default function MyApp() {
  return (
    <div>
      <h1>আমার অ্যাপে তোমাকে স্বাগতম!</h1>
      <MyButton />
    </div>
  );
}
```

এখানে `export default` নামে একটা জিনিস দেখছো, তাই না? আপাতত শুধু জেনে রাখো, এটা দিয়ে বোঝানো হচ্ছে যে `MyApp` কম্পোনেন্টটাই হলো এই ফাইলের প্রধান কম্পোনেন্ট। জাভাস্ক্রিপ্টের এই `export`, `import` ব্যাপারগুলো যদি নতুন মনে হয়, [MDN](https://developer.mozilla.org/en-US/docs/web/javascript/reference/statements/export) বা [javascript.info](https://javascript.info/import-export) ওয়েবসাইট দুটোতে খুব সুন্দর করে বোঝানো আছে। সময় করে দেখে নিতে পারো।

### JSX দিয়ে মার্কআপ লেখা: _Writting markup With JSX_\!

আগের উদাহরণগুলোতে `<button>`, `<div>` এই যে কোডগুলো দেখলে, এগুলোকে বলা হয় _JSX_। এটা দেখতে HTML-এর মতো, কিন্তু আসলে জাভাস্ক্রিপ্টেরই একটা বিশেষ রূপ। JSX ব্যবহার করাটা কিন্তু বাধ্যতামূলক না, তবে বেশিরভাগ React ডেভেলপার এটা ব্যবহার করে, কারণ এটা কোড লিখতে অনেক সুবিধা করে দেয়। অনেকটা ইংরেজিতে কথা বলার মতো সহজ মনে হয়। তবে চিন্তা নেই, আমরা [এক পোস্টে পরে আলোচনা করব সমস্ত টুলসগুলো নিয়ে](/) , সেগুলো সবই JSX বোঝে।

তবে JSX কিন্তু সাধারণ HTML-এর চেয়ে একটু কড়া ধাঁচের। যেমন, তুমি যদি `<br>` ট্যাগ লেখো, JSX বলবে, " এটাকে ঠিক করে বন্ধ করো, `<br />` এভাবে।"। **আরেকটা নিয়ম হলো**, একটা কম্পোনেন্ট থেকে তুমি একসাথে অনেকগুলো JSX ট্যাগ এমনি এমনি রিটার্ন করতে পারবে না। সবগুলোকে একটা বড় "বাক্সে" (মানে, একটা প্যারেন্ট ট্যাগে, যেমন `<div>...</div>` অথবা একটা খালি র‍্যাপার `<>...</>`) ভরে তারপর রিটার্ন করতে হবে।

```javascript {3,6}
function AboutPage() {
  return (
    // এই <>...</> একটা খালি র‍্যাপার
    <>
      <h1>আমার সম্পর্কে</h1>
      <p>
        হ্যালো বন্ধুরা,
        <br />
        সবাই কেমন আছো?
      </p>
    </>
  );
}
```

যদি তোমার কাছে অনেক আগে থেকে লেখা HTML কোড থাকে, আর সেটাকে তুমি JSX-এ পাল্টে নিতে চাও, তাহলে কিছু [অনলাইন কনভার্টার](https://transform.tools/html-to-jsx) আছে, সেগুলো ব্যবহার করতে পারো। কাজটা অনেক সহজ হয়ে যাবে।

### স্টাইল যোগ করা: চলো, একটু সাজাই _Adding Styles_\!

অ্যাপ তো বানালে, এবার একটু রংচং না দিলে কি চলে? React-এ স্টাইল যোগ করাও খুব সহজ। তুমি যখন কোনো HTML ট্যাগে CSS ক্লাস দিতে চাও, তখন `className` অ্যাট্রিবিউটটা ব্যবহার করবে। সাধারণ HTML-এ আমরা `class` লিখি, React-এ সেটার নাম `className`। কাজ কিন্তু একই\!

```javascript
<img className="avatar-image" />
```

আর তারপর, তুমি একটা আলাদা CSS ফাইলে (যেমন `styles.css`) ওই ক্লাসের জন্য স্টাইলগুলো লিখে দেবে:

```css
/* styles.css ফাইলে */
.avatar-image {
  border-radius: 50%; /* ছবিটাকে গোল করে দেবে */
  border: 2px solid skyblue; /* একটা আকাশী বর্ডার দেবে */
}
```

React কিন্তু তোমাকে জোর করবে না যে এভাবেই CSS ফাইল যোগ করতে হবে। অনেক উপায় আছে। সবচেয়ে সহজ হলো, তোমার HTML ফাইলের মধ্যে একটা `<link>` ট্যাগ দিয়ে CSS ফাইলটা যোগ করে দেওয়া। আর যদি তুমি কোনো বিল্ড টুল ( যেমন Vite, Webpack) বা কোনো ফ্রেমওয়ার্ক (যেমন Next.js) ব্যবহার করো, তাহলে তাদের ডকুমেন্টেশনে লেখা থাকবে কীভাবে CSS ফাইল যোগ করতে হয়।

### ডেটা দেখানো: স্ক্রিনে তথ্য ফুটিয়ে তোলা _Displaying Data_

**JSX-এর একটা দারুণ ক্ষমতা হলো**, _এটা তোমাকে জাভাস্ক্রিপ্টের শক্তি সরাসরি মার্কআপের মধ্যে লিখতে দেয়, ব্যবহার করতে দেয়।_ কীভাবে? কার্লি ব্রাকেট দিয়ে `{}` দিয়ে\! ধরো, তোমার একটা জাভাস্ক্রিপ্ট ভ্যারিয়েবল আছে, আর তুমি সেটার মান স্ক্রিনে দেখাতে চাও। এই `{}` ব্র্যাকেট দুটো হলো তোমার সেই জাদুর কাঠি।

যেমন, ধরো `user` নামে একটা অবজেক্ট আছে, যার ভেতরে `name` নামে একটা প্রপার্টি আছে। সেটা দেখাতে হলে:

```javascript {3}
return (
  <h1>
    {user.name} {/* user অবজেক্টের name প্রপার্টির মান এখানে দেখাবে */}
  </h1>
);
```

শুধু টেক্সট না, তুমি চাইলে JSX ট্যাগের অ্যাট্রিবিউটের মানও এভাবে জাভাস্ক্রিপ্ট থেকে দিতে পারো। তবে এখানে কোটেশন `""` এর বদলে `{}` ব্যবহার করতে হবে। যেমন, `className="avatar"` লিখলে `"avatar"` স্ট্রিংটাই ক্লাস হিসেবে বসবে, **এই ক্ষেত্রে _Template String_ ব্যবহার করতে হবে।** কিন্তু`src={user.imageUrl}`লিখলে, React বুঝবে যে`user.imageUrl`একটা জাভাস্ক্রিপ্ট ভ্যারিয়েবল, সেটার মানটা বের করে`src` অ্যাট্রিবিউটে বসিয়ে দেবে।

```javascript {3,4}
return (
  <img
    className="avatar"
    src={user.imageUrl} {/* user.imageUrl এর মান এখানে বসবে */}
  />
);
```

এমনকি এই `{}` এর ভেতরে তুমি আরও জটিল জাভাস্ক্রিপ্ট এক্সপ্রেশনও লিখতে পারো। যেমন, দুটো স্ট্রিং যোগ করা:

```javascript
const user = {
  name: "Hedy Lamarr",
  imageUrl: "https://i.imgur.com/yXOvdOSs.jpg",
  imageSize: 90,
};

export default function Profile() {
  return (
    <>
      <h1>{user.name}</h1>
      <img
        className="avatar"
        src={user.imageUrl}
        alt={"Photo of " + user.name} // এখানে স্ট্রিং যোগ করা হয়েছে
        style={{
          // ইনলাইন স্টাইল, একটা জাভাস্ক্রিপ্ট অবজেক্ট হিসেবে
          width: user.imageSize,
          height: user.imageSize,
        }}
      />
    </>
  );
}
```

```css
/* styles.css */
.avatar {
  border-radius: 50%;
}
```

ওপরের কোডে `style={{}}` দেখেছো? এটা কিন্তু কোনো বিশেষ সিনট্যাক্স নয়। `style={...}` এর ভেতরের `{}` দিয়ে একটা জাভাস্ক্রিপ্ট অবজেক্ট বোঝানো হচ্ছে। যখন তোমার স্টাইলগুলো জাভাস্ক্রিপ্ট ভ্যারিয়েবলের উপর নির্ভর করবে, তখন এভাবে `style` অ্যাট্রিবিউট ব্যবহার করা খুব কাজের।

### শর্ত দিয়ে কিছু দেখানো _Conditional Rendering_

অনেক সময়ই আমাদের দরকার হয় যে, একটা নির্দিষ্ট শর্ত পূরণ হলে কিছু একটা দেখাবো, আর শর্ত পূরণ না হলে অন্য কিছু দেখাবো, অথবা কিছুই দেখাবো না। যেমন, ইউজার যদি লগইন করা থাকে, তাহলে তার নাম দেখাবে, না থাকলে লগইন বাটন দেখাবে।

খুশির খবর হলো, এটা করার জন্য React-এ তোমাকে নতুন কিছু শিখতে হবে না\! তুমি জাভাস্ক্রিপ্টে যেভাবে `if...else` স্টেটমেন্ট ব্যবহার করো, ঠিক সেভাবেই এখানেও করতে পারো:

```javascript
let content;
if (isLoggedIn) {
  content = <AdminPanel />;
} else {
  content = <LoginForm />;
}
return (
  <div>
    {content} {/* content ভ্যারিয়েবলের মান এখানে বসবে */}
  </div>
);
```

যদি তুমি কোডটা আরও একটু ছোট আর স্মার্টভাবে লিখতে চাও, তাহলে কন্ডিশনাল `?` অপারেটর ( এটাকে টারনারি অপারেটরও বলে) ব্যবহার করতে পারো। `if`-এর মতো এটা JSX-এর বাইরে লিখতে হয় না, সরাসরি JSX-এর ভেতরেই ব্যবহার করা যায়:

```javascript
<div>
  {isLoggedIn ? ( // যদি isLoggedIn সত্যি হয়
    <AdminPanel /> // তাহলে এটা দেখাও
  ) : (
    // নইলে (মিথ্যা হলে)
    <LoginForm /> // এটা দেখাও
  )}
</div>
```

আর যদি এমন হয় যে, শর্ত পূরণ হলে কিছু একটা দেখাবে, কিন্তু শর্ত পূরণ না হলে কিছুই দেখানোর দরকার নেই (`else` অংশটা লাগবে না), তাহলে আরও ছোট করে লজিক্যাল `&&` অপারেটর দিয়েও কাজ সারা যায়:

```javascript
<div>
  {isLoggedIn && <AdminPanel />}{" "}
  {/* isLoggedIn সত্যি হলেই <AdminPanel /> দেখাবে */}
</div>
```

এই সবগুলো পদ্ধতি কিন্তু অ্যাট্রিবিউটের মান শর্তসাপেক্ষে বসানোর জন্যও ব্যবহার করা যায়। যদি এই `?` বা `&&` অপারেটরগুলো তোমার কাছে নতুন মনে হয়, কোনো চিন্তা নেই\! তুমি শুরুতেই `if...else` দিয়েই কাজ চালাতে পারো। আস্তে আস্তে সব সহজ হয়ে যাবে।

### তালিকা ধরে অনেক কিছু দেখানো (List Rendering)

ধরো, তোমার কাছে অনেকগুলো বন্ধুর নামের একটা তালিকা আছে, আর তুমি এই প্রত্যেকটা নাম স্ক্রিনে দেখাতে চাও। এইরকম তালিকা ধরে অনেকগুলো জিনিস দেখানোর জন্য তুমি জাভাস্ক্রিপ্টের পরিচিত লুপ (যেমন `for` লুপ) অথবা অ্যারের `map()` ফাংশন ব্যবহার করতে পারো। React-এর জগতে `map()` ফাংশনটা খুবই জনপ্রিয়।

মনে করো, তোমার কাছে কিছু পণ্যের একটা অ্যারে (তালিকা) আছে:

```javascript
const products = [
  { title: "ফুলকপি", id: 1 },
  { title: "রসুন", id: 2 },
  { title: "পেঁপে", id: 3 },
];
```

এখন তোমার কম্পোনেন্টের ভেতরে, তুমি `map()` ফাংশনটা ব্যবহার করে এই `products` অ্যারের প্রত্যেকটা আইটেমের জন্য একটা করে `<li>` আইটেম তৈরি করতে পারো:

```javascript
const listItems = products.map((product) => (
  // প্রত্যেকটা product এর জন্য একটা li তৈরি হচ্ছে
  <li key={product.id}>
    {" "}
    {/* key জিনিসটা খুব জরুরি, বলছি কেন */}
    {product.title}
  </li>
));

return <ul>{listItems}</ul>;
```

একটা জিনিস খুব ভালো করে খেয়াল করো: প্রত্যেকটা `<li>` ট্যাগের সাথে একটা `key` নামের অ্যাট্রিবিউট আছে। যখন তুমি কোনো তালিকা দেখাবে, তখন তালিকার প্রত্যেকটা আইটেমের জন্য এমন একটা `key` দিতে হবে, যেটার মান (স্ট্রিং বা নম্বর) ওই তালিকার অন্য আইটেমগুলো থেকে আলাদা হবে। এই `key` জিনিসটা খুব জরুরি। React এটা দিয়ে বুঝতে পারে তালিকার কোন আইটেমটা কোনটা, অনেকটা ক্লাসরুমে প্রত্যেক ছাত্রের একটা রোল নম্বরের মতো। তুমি যদি পরে এই তালিকা থেকে কিছু বাদ দাও, নতুন কিছু যোগ করো, বা আইটেমগুলোর ক্রম বদলাও, তখন React এই `key` দেখেই বুঝতে পারে যে ঠিক কী পরিবর্তন হয়েছে। সাধারণত, এই `key`-এর মান তোমার ডেটা থেকেই আসে, যেমন ডাটাবেসের কোনো আইডি।

চলো, একটা উদাহরণ দেখি:

```javascript
const products = [
  { title: "ফুলকপি", isFruit: false, id: 1 },
  { title: "রসুন", isFruit: false, id: 2 },
  { title: "পেঁপে", isFruit: true, id: 3 },
];

export default function ShoppingList() {
  const listItems = products.map((product) => (
    <li
      key={product.id}
      style={{
        // ফলের রঙ হবে ম্যাজেন্টা, সবজির রঙ হবে সবুজ
        color: product.isFruit ? "magenta" : "darkgreen",
      }}
    >
      {product.title}
    </li>
  ));

  return <ul>{listItems}</ul>;
}
```

### ইউজারের কাজে সাড়া দেওয়া _Responding to events_

তোমার অ্যাপ তো শুধু দেখানোর জন্য নয়, ইউজার তো সেটার সাথে নানা কাজও করবে, তাই না? যেমন, বাটনে ক্লিক করবে, ফর্মে কিছু লিখবে ইত্যাদি। এই যে ইউজারের বিভিন্ন কাজ, এগুলোকে বলে **ইভেন্ট**। আর এই ইভেন্ট ঘটলে তুমি যে কাজটা করতে চাও, সেটা একটা ফাংশনের ভেতর লিখে রাখো। এই ফাংশনগুলোকে বলে **ইভেন্ট হ্যান্ডলার**।

React কম্পোনেন্টের ভেতরেই তুমি এই ইভেন্ট হ্যান্ডলার ফাংশনগুলো লিখে ফেলতে পারো:

```javascript {2-4,7}
function MyButton() {
  // এটা হলো আমাদের ইভেন্ট হ্যান্ডলার ফাংশন
  function handleClick() {
    alert("তুমি আমাকে ক্লিক করেছো!");
  }

  return (
    // onClick ইভেন্টে handleClick ফাংশনটা জুড়ে দিলাম
    <button onClick={handleClick}>আমাকে ক্লিক করো</button>
  );
}
```

এখানে একটা মজার জিনিস খুব ভালো করে খেয়াল করো: `onClick={handleClick}` লেখার সময় `handleClick` এর পর কিন্তু `()` প্রথম বন্ধনীটা দিইনি\! কারণ আমরা ফাংশনটাকে তখনি কল করতে চাই না। বরং আমরা React-কে বলে দিচ্ছি যে, "যখন এই বাটনটায় ক্লিক করা হবে, তখন তুমি এই `handleClick` ফাংশনটা কল কোরো।" যদি তুমি `handleClick()` এভাবে লিখতে, তাহলে কম্পোনেন্টটা স্ক্রিনে দেখানোর সময়ই ফাংশনটা কল হয়ে যেত, ক্লিকের জন্য অপেক্ষাই করতো না\!

### স্ক্রিনে বদলে ফেলা _Updating the screen_

অনেক সময় এমন দরকার হয় যে, তোমার কম্পোনেন্টকে কিছু তথ্য "মনে রাখতে" হবে এবং সেই অনুযায়ী স্ক্রিনে কিছু দেখাতে হবে। যেমন ধরো, একটা বাটন কতবার ক্লিক করা হলো, সেই সংখ্যাটা তুমি গুনে রাখতে চাও আর দেখাতে চাও। এই "মনে রাখা"র কাজটা করার জন্য React আমাদের একটা দারুণ জিনিস দিয়েছে, যার নাম হলো _স্টেট (state)_।

স্টেট ব্যবহার করার জন্য, প্রথমে তোমাকে React থেকে `useState` নামে একটা বিশেষ ফাংশন (এটাকে হুক বলে, একটু পরেই বলছি) ইম্পোর্ট করে নিতে হবে:

```javascript
import { useState } from "react";
```

তারপর, তোমার কম্পোনেন্টের ভেতরে তুমি একটা _স্টেট ভ্যারিয়েবল_ ডিক্লেয়ার করতে পারো:

```javascript
function MyButton() {
  // এই দেখো, useState ব্যবহার করে count নামে একটা স্টেট তৈরি করলাম
  // এর প্রাথমিক মান দিলাম 0
  const [count, setCount] = useState(0);
  // ...
```

**`useState` তোমাকে দুটো জিনিস দেবে:**

<hr />
1. তোমার বর্তমান স্টেট-এর মানটা (আমাদের উদাহরণে `count`, যার প্রাথমিক মান `0`)।

2. একটা ফাংশন, যেটা দিয়ে তুমি ওই স্টেট-এর মান বদলাতে পারবে (আমাদের উদাহরণে `setCount`)।

তুমি এগুলোর যেকোনো নাম দিতে পারো, তবে সাধারণত সবাই `[একটা নাম, setসেইনাম]` (যেমন: `[name, setName]`, `[age, setAge]`) এই রকম প্যাটার্ন ব্যবহার করে।

প্রথমবার যখন তোমার `MyButton` কম্পোনেন্টটা স্ক্রিনে দেখানো হবে, তখন `count` এর মান হবে `0`, কারণ তুমি `useState(0)` এভাবে তাকে শুরুতেই `0` দিয়েছো। যখন তুমি এই `count` এর মান বদলাতে চাইবে, তখন `setCount` ফাংশনটা কল করবে আর নতুন মানটা তাকে বলে দেবে। যেমন, নিচের বাটনটা ক্লিক করলে `count` এর মান এক এক করে বাড়তে থাকবে:

```javascript {5}
function MyButton() {
  const [count, setCount] = useState(0); // স্টেট

  function handleClick() {
    // setCount দিয়ে count এর মান ১ বাড়িয়ে দিলাম
    setCount(count + 1);
  }

  return (
    <button onClick={handleClick}>
      ক্লিক হয়েছে {count} বার {/* এখানে count এর বর্তমান মান দেখাবে */}
    </button>
  );
}
```

যখন তুমি `setCount` কল করে `count`-এর মান বদলাবে, React সাথে সাথে তোমার কম্পোনেন্ট ফাংশনটাকে আবার কল করবে (মানে, রি-রেন্ডার করবে)। তখন `count`-এর নতুন মান (`1`, তারপর `2`, এভাবে) স্ক্রিনে আপডেট হয়ে যাবে। দারুণ না ব্যাপারটা?

যদি তুমি একই `MyButton` কম্পোনেন্টটা কয়েকবার স্ক্রিনে দেখাও, তাহলে প্রত্যেকটা `MyButton` তার নিজের `count` স্টেট আলাদাভাবে মনে রাখবে। একটা বাটনের `count` বাড়ালে অন্য বাটনের `count`-এর উপর কোনো প্রভাব পড়বে না। বিশ্বাস না হলে, নিচের কোডটা চালিয়ে দুইটি বাটনে ক্লিক করে দেখো:

```javascript
import { useState } from "react";

export default function MyApp() {
  return (
    <div>
      <h1>আলাদা আলাদা কাউন্টার</h1>
      <MyButton /> {/* এই বাটনের স্টেট আলাদা */}
      <MyButton /> {/* এই বাটনের স্টেটও আলাদা */}
    </div>
  );
}

function MyButton() {
  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return <button onClick={handleClick}>ক্লিক হয়েছে {count} বার</button>;
}
```

```css
/* styles.css */
button {
  display: block;
  margin-bottom: 5px;
}
```

দেখলে তো, প্রত্যেকটা বাটন কেমন নিজের হিসেব নিজে রাখছে\!

### হুকস (Hooks) ব্যবহার করা: _Using Hooks_\!

একটু আগে `useState`-এর কথা বললাম না? এই রকম `use` দিয়ে শুরু হওয়া ফাংশনগুলোকে React-এর ভাষায় বলে _হুকস (Hooks)_। `useState` হলো React-এর নিজের দেওয়া (বিল্ট-ইন) একটা হুক। এরকম আরও কিছু বিল্ট-ইন হুক আছে, যেগুলো তুমি [API রেফারেন্সে](https://react.dev/reference/react/hooks) খুঁজে পাবে।

> তুমি চাইলে নিজেও কিন্তু একাধিক হুক মিলিয়ে নতুন কাস্টম হুক তৈরি করতে পারো\!

হুকসগুলো সাধারণ ফাংশনের চেয়ে একটু বেশি নিয়ম মেনে চলে। তুমি হুকস কল করতে পারবে শুধুমাত্র তোমার কম্পোনেন্ট ফাংশনের একদম শুরুতে, অথবা অন্য কোনো কাস্টম হুকের ভেতরে। _কোনো লুপ বা কন্ডিশনের ভেতরে সরাসরি হুক কল করা যায় না।_ যদি এমন দরকার হয়, তাহলে ওই অংশটুকুর জন্য একটা নতুন কম্পোনেন্ট বানিয়ে, তার ভেতরে হুকটা ব্যবহার করতে পারো।

### এক কম্পোনেন্টের তথ্য অন্য কম্পোনেন্টে দেওয়া _Sharing data between Components_

আগের উদাহরণে আমরা দেখলাম, প্রত্যেকটা `MyButton` কম্পোনেন্টের `count` ছিল যার যার নিজের। একটা বাটনে ক্লিক করলে শুধু সেই বাটনের `count` পরিবর্তন হচ্ছিল।

কিন্তু অনেক সময়ই এমন দরকার হয় যে, দুটো বা তার বেশি কম্পোনেন্ট একই ডেটা ব্যবহার করবে এবং সেই ডেটা পরিবর্তন হলে সবাই একসাথেই আপডেট হবে।

ধরো, তুমি চাও যে দুটো `MyButton` কম্পোনেন্টই একই `count` দেখাক এবং যেকোনো একটাতে ক্লিক করলেই দুটোতেই `count` আপডেট হয়ে যাক। এটা করার জন্য, তোমাকে ওই `count` স্টেটটাকে প্রত্যেকটা বাটন থেকে বের করে এনে, তাদের সবার সাধারণ প্যারেন্ট কম্পোনেন্টে (মানে, যে কম্পোনেন্টের ভেতরে ওই বাটনগুলো আছে) নিয়ে যেতে হবে। এই পদ্ধতিকে বলে "লিফটিং স্টেট আপ" (Lifting State Up) বা "স্টেট উপরে তুলে আনা"।

আমাদের উদাহরণের ক্ষেত্রে, `MyButton` গুলোর প্যারেন্ট কম্পোনেন্ট হলো `MyApp`। তাহলে, চলো স্টেটটাকে `MyApp`-এ নিয়ে যাই:

এই পদ্ধতিতে, `MyApp` কম্পোনেন্ট `count` স্টেট এবং সেটি পরিবর্তনের লজিক (যেমন `handleClick` ফাংশন) নিজের কাছে রাখবে। এরপর, এই স্টেট এবং লজিক সে তার চাইল্ড কম্পোনেন্টগুলোতে (`MyButton`) পাস করে দেবে।

প্রথমে, `MyButton` থেকে স্টেটটাকে বের করে `MyApp`-এ নিয়ে আসি:

```javascript {2-6,18}
export default function MyApp() {
  // স্টেট এখন MyApp এর কাছে
  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return (
    <div>
      <h1>আলাদা আলাদা কাউন্টার</h1> {/* এটাকে পরে পাল্টে দেবো */}
      <MyButton />
      <MyButton />
    </div>
  );
}

function MyButton() {
  // MyButton থেকে useState এবং handleClick সরিয়ে নিয়েছি
}
```

এরপর, `MyApp` থেকে `count` স্টেট এবং `handleClick` ফাংশনটাকে প্রত্যেকটা `MyButton`-এ পাস করে দিতে হবে। একটা কম্পোনেন্ট থেকে তার চাইল্ড কম্পোনেন্টে তথ্য পাঠানোর এই উপায়কে বলে _props_ (প্রপস)। অনেকটা বাবা-মা যেমন সন্তানদের বিভিন্ন জিনিস দেয়, সেরকম। JSX-এর কার্লি ব্রেস `{}` ব্যবহার করেই আমরা এই props পাস করতে পারি:

```javascript {11-12}
export default function MyApp() {
  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return (
    <div>
      <h1>একসাথে আপডেট হওয়া কাউন্টার</h1>
      {/* count এবং handleClick কে props হিসেবে MyButton এ পাঠাচ্ছি */}
      <MyButton count={count} onClick={handleClick} />
      <MyButton count={count} onClick={handleClick} />
    </div>
  );
}
```

সবশেষে, `MyButton` কম্পোনেন্টটাকে এমনভাবে পরিবর্তন করতে হবে যেন সে তার প্যারেন্ট (`MyApp`) থেকে পাঠানো এই `count` আর `onClick` props গুলো পড়তে এবং ব্যবহার করতে পারে:

```javascript {1,3}
// MyButton এখন count আর onClick props গ্রহণ করছে
function MyButton({ count, onClick }) {
  return (
    // onClick prop এ থাকা ফাংশনটা এখানে কল হবে
    // count prop থেকে মানটা এখানে দেখাবে
    <button onClick={onClick}>ক্লিক হয়েছে {count} বার</button>
  );
}
```

এখন কী হবে দেখো\! যখন তুমি যেকোনো একটা বাটনে ক্লিক করবে, তখন ওই বাটনের `onClick` প্রপ-এ থাকা `handleClick` ফাংশনটা (যেটা `MyApp`-এর ভেতরে লেখা) কল হবে। সেই ফাংশন `setCount(count + 1)` কল করে `MyApp`-এর `count` স্টেটটা আপডেট করে দেবে। যেই `MyApp`-এর `count` স্টেট আপডেট হবে, React সাথে সাথে `MyApp` এবং তার চাইল্ড `MyButton` কম্পোনেন্টগুলোকে রি-রেন্ডার করবে। তখন `count`-এর নতুন মানটা props হিসেবে দুটো `MyButton`-এই চলে যাবে, আর দুটো বাটনেই নতুন `count` দেখাবে\!

এভাবেই "স্টেট উপরে তুলে আনার" মাধ্যমে আমরা বিভিন্ন কম্পোনেন্টের মধ্যে ডেটা শেয়ার করতে পারি।

চলো, পুরো কোডটা একসাথে দেখে নিই:

```javascript
import { useState } from "react";

export default function MyApp() {
  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return (
    <div>
      <h1>একসাথে আপডেট হওয়া কাউন্টার</h1>
      <MyButton count={count} onClick={handleClick} />
      <MyButton count={count} onClick={handleClick} />
    </div>
  );
}

function MyButton({ count, onClick }) {
  return <button onClick={onClick}>ক্লিক হয়েছে {count} বার</button>;
}
```

```css
/* styles.css */

  display: block;
  margin-bottom: 5px;
}
```

### এখন কি করবে?

ব্যস, তুমি কিন্তু React-এর অনেকখানি শেখার পথে চলে এসেছো। যা যা শিখলে – কম্পোনেন্ট, JSX, props, state, event handling – এগুলোই হলো React-এর মূল ভিত্তি। এগুলো ভালো করে বুঝতে পারলে, তোমার React যাত্রা অনেক সহজ হয়ে যাবে।

তবে শুধু পড়লেই তো হবে না, নিজে হাতে কোড না করলে ঠিক মজাটা আসবে না। তাই আমার পরামর্শ হলো, তুমি [React-এর অফিশিয়াল টিউটোরিয়ালটা](https://react.dev/learn/tutorial-tic-tac-toe) একবার করে ফেলো। সেখানে তুমি এই সবকিছু ব্যবহার করে নিজেই একটা ছোটখাটো টিক-ট্যাক-টো গেম বানিয়ে ফেলতে পারবে\!
